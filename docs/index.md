# Стандарты, законы и Docs-as-Code

Я помогаю превращать требования ГОСТ и законов в понятные документы, а код — в живую документацию.

Привожу документацию в соответствие с жесткими требованиями **ЕСКД (ГОСТ 2.ххх)** и **ЕСПД (ГОСТ 19.ххх)**. 

В пет-проектах внедряю современный подход **Docs-as-Code**: веду документацию в Markdown, автоматизирую сборку статических сайтов (MkDocs) и настраиваю публикацию через GitHub Actions.

## Мои компетенции

<div class="grid cards" markdown>

- :material-file-document-check-outline:{ .lg .middle } **Проектная документация**

    ---
  - Разработка и оформление документов по **ГОСТ 2.ххх** (конструкторская) и **ГОСТ 19.ххх** (программная).
  - Подготовка проектов нормативно-правовых актов (федеральные законы, ПП РФ) в сфере космической деятельности.
  - Прохождение нормоконтроля документации.
  - Защита проектной документации на каждом этапе.

- :material-code-tags:{ .lg .middle } **Docs-as-Code (пет-проекты)**

    ---
  - Ведение документации в репозитории (**Markdown**).
  - Генерация статических сайтов (**MkDocs, Material for MkDocs**).
  - Настройка CI/CD пайплайнов для автоматической сборки и публикации документации при пушах в репозиторий (**GitHub Actions**).
  - Интеграция с кодом: работа с **docstrings**, поддержание актуальности документации API при изменении кода.

</div>
